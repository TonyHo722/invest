1. Screener Helper Syntax Error

During execution, the screener failed with SyntaxError: 'continue' not properly in loop on line 199. This error occurred because the filtering check was refactored into a helper function (check_candidate) which is submitted to a thread pool executor, rather than executing in a direct loop.

Why it failed:
The continue statement is only valid inside loop structures (e.g., for or while). Inside a function body that runs asynchronously, it raises a compile-time syntax error.
BEFORE (Buggy code)
def check_candidate(c):
    try:
        ...
        pct_diff = ((c['price'] / c['dma200']) - 1) * 100
        
        # Exclude stocks with extreme anomalies
        if pct_diff <= -80.0:
            continue  # <-- Raised SyntaxError!
            
        results.append(...)
    except Exception:
        ...
AFTER (Fixed code)
def check_candidate(c):
    try:
        ...
        pct_diff = ((c['price'] / c['dma200']) - 1) * 100
        
        # Exclude stocks with extreme anomalies
        if pct_diff <= -80.0:
            return  # <-- Correctly exits function execution
            
        results.append(...)
    except Exception:
        ...

2. Sub-optimal Split-Factor Selection

During the automatic split-adjustment phase, Chikuho Bank (7173.T) experienced an 8-for-1 forward split. Its daily price dropped from 11,690.00 JPY to 1,528.75 JPY, which translates to a ratio of 7.647. Under the original split detection algorithm, it was incorrectly matched as a 7:1 split instead of 8:1.

Why it selected 7:1:
The original algorithm checked split factor candidates sequentially. Since abs(7.647 - 7) / 7 = 9.2% (which is below the 15% threshold), the check for 7 succeeded and the loop immediately broke, ignoring the closer match of 8 (error: 4.4%).
BEFORE (Sequential Break)
# Checks factors sequentially; breaks at first match
for R in factors:
    err = abs(ratio - R) / R
    if err < 0.15:
        split_factor = R
        break  # Selected 7 immediately
AFTER (Minimum Error Selection)
# Scans all factors to find the absolute minimum error
best_R = None
min_err = 999.0
for R in factors:
    err = abs(ratio - R) / R
    if err < 0.15 and err < min_err:
        min_err = err
        best_R = R
if best_R is not None:
    split_factor = best_R  # Correctly selects 8
Effect of the Fix:
Chikuho Bank (7173.T) now has its history accurately split-adjusted by 8:1. This logic is generic and ensures all stocks undergoing unrecorded splits are adjusted using the mathematically closest integer split factor.
